# UI/UXデザインガイドライン

本プロジェクト（イケダンラボ）のページ構成・修正時に必ず参照するルール集。
UX心理学の法則に基づき、20代前半〜30代前半男性のアフィリエイトブログに最適化している。

> 参考: [UX心理学 単語集 | 松下村塾](https://www.shokasonjuku.com/ux-psychology)

---

## 1. ナビゲーション・レイアウトのルール

### 1-1. 標準的な配置パターンを守る（ヤコブの法則・親近性バイアス・認知容易性）

ユーザーは他サイトで学んだ操作パターンをそのまま期待する。独自レイアウトは混乱を招く。

| 要素 | 配置ルール |
|------|-----------|
| ヘッダー | ページ最上部に固定（sticky）。左にロゴ、右にナビゲーション |
| カテゴリナビ | ヘッダー内またはヘッダー直下。3カテゴリを常に見せる |
| パンくずリスト | 記事ページ上部、タイトルの前に配置 |
| 目次（TOC） | 記事本文の直前。折りたたみ不可（常に表示） |
| CTAボタン | 記事末尾のアフィリエイトセクション。記事途中にも1箇所まで可 |
| フッター | サイト最下部。リンク一覧とコピーライト |

### 1-2. F型の視線動線を意識する（F型視線誘導の法則）

ユーザーの視線はページ左上から「F」の形に動く。

- **重要な情報（見出し・CTA・カテゴリ名）は左寄せ**にする
- 記事一覧ページでは、最も訴求力のある記事を**左上（最初の位置）**に配置する
- 右カラムやサイドバーに重要情報を置かない（本プロジェクトは1カラム構成を維持する）

### 1-3. モバイルファーストの導線設計

- タップターゲットは最低 **44px × 44px** を確保する
- カテゴリタブは横スクロール可能にし、3つすべてが初期表示で見えるようにする
- ハンバーガーメニューは右上に固定し、アイコンの意味が明確に伝わるデザインにする（アフォーダンス）

## 2. 情報設計・認知負荷のルール

### 2-1. 選択肢は最小限にする（ヒックの法則・決断疲れ）

選択肢が増えると意思決定に時間がかかり、離脱につながる。

- カテゴリは**4つ**に留める（現状: メンズスキンケア、脱毛、AGA・薄毛、食事・健康法）
- 記事一覧の1ページあたりの表示件数は**最大12件**。それ以上はページネーション
- アフィリエイトボックスは1記事あたり**3つ以内**。最もおすすめの1つを目立たせる
- 比較表の項目は**5列以内**に収める

### 2-2. 段階的に情報を開示する（段階的開示・認知負荷）

ユーザーに一度に大量の情報を見せない。

- 記事冒頭に目次を置き、全体構造を先に把握させる
- 比較表は最初に結論（おすすめ）を示し、詳細は本文で段階的に説明する
- 長い記事（3,000文字超）では、h2見出しごとに情報を区切り、視覚的な休憩ポイントを作る
- FAQや補足情報は記事末尾にまとめ、本文の流れを止めない

### 2-3. 入力フォームの負担を最小化する（ロバストネス原則・認知負荷）

将来的に検索機能やお問い合わせフォームを実装する場合:

- 入力欄は必要最小限にする
- プレースホルダーで入力例を示す
- 全角・半角の違いはシステム側で吸収する

## 3. 視覚的階層・注意誘導のルール

### 3-1. 重要な要素を系列の最初と最後に置く（系列位置効果）

リストやナビゲーションでは、最初と最後の要素が記憶に残りやすい。

- カテゴリタブ: 注力度が高いカテゴリを**最初**に配置する
- アフィリエイト比較表: 最もおすすめの商品を**1行目**に、次点を**最終行**に配置する
- 記事一覧: 最も読ませたい記事をグリッドの**最初（左上）**に配置する

### 3-2. CTAを孤立させて目立たせる（孤立効果・コントラスト効果）

周囲と異なる要素は記憶に残る。CTAは意図的に「浮かせる」。

- CTAボタンの色は `--color-cta`（オレンジ系）を使い、他のUI要素と差別化する
- アクセントカラー `--color-accent`（ティール系）はリンクやナビゲーションに使い、CTAとは明確に分ける
- **1画面に表示するCTAは1つだけ**。複数のCTAが同時に見えると効果が薄れる
- CTAボタンには `box-shadow` とホバー時の `translateY` で立体感を出す（アフォーダンス）

### 3-3. 色の意味を一貫させる（ストループ効果）

色と意味の不一致は認知の遅延を招く。

| 色 | 用途 | CSS変数 |
|----|------|---------|
| オレンジ系 | CTA・行動を促す要素 | `--color-cta`, `--color-cta-hover` |
| ティール系 | リンク・ナビゲーション | `--color-accent`, `--color-accent-hover` |
| グレー系 | 補足情報・メタデータ | `--color-text-muted`, `--color-text-secondary` |
| 黄色背景 | バッジ・ラベル | `--color-badge-bg`, `--color-badge` |

- エラー表示には赤、成功表示には緑を使う（社会的慣習に従う）
- 注意喚起の赤をCTAに使わない（エラーと混同される）

### 3-4. 角丸で親しみやすさを出す（輪郭線バイアス）

ターゲット層（20代〜30代男性）にカジュアルで親しみやすい印象を与える。

- カード要素: `border-radius: 12px`
- ボタン: `border-radius: 8px`
- バッジ・タグ: `border-radius: 100px`（ピル型）
- 目次・引用ブロック: `border-radius: 6px〜8px`
- 警告やエラーには角丸を小さくし、緊張感を適度に持たせる

## 4. 行動促進・コンバージョンのルール

### 4-1. 押し売り感のないCTA設計（フレーミング効果・損失回避）

ターゲット層は押し売りに敏感。「背中を押す」導線を心がける。

**CTAテキストのルール:**

| NG例 | OK例 | 理由 |
|------|------|------|
| 「今すぐ購入！」 | 「公式サイトで詳しく見る」 | 情報収集段階の読者に合わせる |
| 「残りわずか！急いで！」 | 「無料カウンセリングを予約する」 | 過度な希少性の演出は不信感を招く |
| 「絶対おすすめ！」 | 「Amazonで価格を見る」 | 客観的な表現で信頼感を保つ |

- CTAの前に**感情の代弁**を置く（「もっと早く知りたかった」等）
- 価格表示は「月3,000円台〜」のように**ポジティブなフレーム**で提示する
- 無料要素があれば「無料カウンセリング」「初月0円」等を明示する（損失回避: 失うものがないことを強調）

### 4-2. 社会的証明を活用する（社会的証明）

他者の行動や評価は意思決定を後押しする。

- アフィリエイトボックスのバッジに「人気No.1」「メンズ人気No.1」等を表示する
- 比較表に「利用者数」「口コミ評価」の列を設ける
- 体験談ベースの語り口で、著者自身のレビューを共感的に伝える
- **偽のレビューや誇張表現は絶対に使わない**（信頼を致命的に損なう）

### 4-3. 進捗と達成感を可視化する（目標勾配効果・ツァイガルニク効果）

ゴールに近づくほどモチベーションが上がる。

- 記事ページの**読了プログレスバー**は必ず表示する（現在実装済み: `ReadingProgress` コンポーネント）
- 記事冒頭に「読了時間: 約○分」を表示し、ゴールを明示する
- 比較記事では「3つのポイントを確認するだけ」等、ステップ数を先に示す
- 目次でチェックリスト的な構造を見せ、「まだ読んでいない部分がある」と感じさせる

### 4-4. ターゲットに刺さる言葉を使う（カクテルパーティー効果）

自分に関係があると感じた情報には自然と注意が向く。

- 記事タイトルに**「メンズ」「男性」「20代」**等のペルソナワードを含める
- 悩みを具体的に言語化する（「テカリが気になる」「ヒゲ剃りが面倒」等）
- 抽象的な表現（「おすすめ○選」のみ）ではなく、**悩みベース**のタイトルにする

## 5. 信頼性・ブランドのルール

### 5-1. 一貫したデザイントークンを守る（単純接触効果・認知容易性）

同じデザインパターンに繰り返し接触することで信頼感が醸成される。

- カード・ボタン・バッジ等のスタイルは `globals.css` のCSS変数とBEM命名規則を必ず使う
- 新しいコンポーネントを追加する場合も、既存の変数・パターンを流用する
- フォントは `--font-heading` / `--font-body` / `--font-mono`（JetBrains Mono: ラボ感演出用）を使用。`--font-mono` はメタ情報・ラベル・バッジ等のアクセントに限定し、本文には使用しない
- 色はCSS変数で定義されたものだけを使う。ハードコーディングしない

### 5-2. 読みやすさを最優先する（認知容易性・主題内容効果）

アフィリエイトブログの生命線は「読了されること」。

- 本文の `line-height` は **1.75〜1.9** を維持する
- 段落間のマージンは **24px** を基本とし、詰めすぎない
- h2見出しは背景色付きで**視覚的な区切り**を明確にする
- h3見出しは左ボーダーで**サブセクション**であることを視覚的に示す
- 1段落あたり **3〜4行**（モバイルで5行以内）を目安にする
- 専門用語には初出時に補足説明を入れる（「AGA（男性型脱毛症）」等）

### 5-3. 表示速度を体感させる（ドハティの閾値）

システムの応答が400ms以内であれば、ユーザーは「速い」と感じる。

- ISR（60秒再検証）で実質的な静的配信を維持する
- 画像は microCMS の変換パラメータ（`?w=800` 等）で適切なサイズに最適化する
- ページ遷移時にローディング状態を示す（将来的には `loading.tsx` を活用）
- アニメーションの `transition` は **0.15s〜0.2s** に留め、軽快さを保つ

## 6. ページ種別ごとの適用チェックリスト

### トップページ (`/`)

- [ ] ファーストビューに最も訴求力のある記事が表示されている（系列位置効果）
- [ ] カテゴリタブが3つ表示され、注力カテゴリが最初にある（ヒックの法則）
- [ ] 記事カードのグリッドが整然と並び、情報量が均一である（認知負荷軽減）
- [ ] ヘッダーのナビゲーションが標準的な位置にある（ヤコブの法則）

### 記事詳細ページ (`/articles/[slug]`)

- [ ] パンくずリストが記事上部にある（認知容易性）
- [ ] 目次が本文前に表示され、全体構造が把握できる（段階的開示）
- [ ] 読了プログレスバーが表示されている（目標勾配効果）
- [ ] 見出し（h2/h3）が視覚的に区別されている（選択的注意）
- [ ] CTAボタンが周囲と色で差別化されている（孤立効果・コントラスト効果）
- [ ] CTAテキストが押し売りでなく、情報提供型になっている（フレーミング効果）
- [ ] アフィリエイトボックスが3つ以内である（ヒックの法則）
- [ ] 関連記事セクションが記事末尾にある（ツァイガルニク効果）

### カテゴリ一覧ページ (`/category/[categoryId]`)

- [ ] カテゴリ名と記事件数がページ上部に表示されている（認知容易性）
- [ ] 記事カードが統一されたフォーマットで並んでいる（認知負荷軽減）
- [ ] 最も読ませたい記事がグリッドの最初にある（系列位置効果・F型視線）
